# AutoResearch Loop Template

![logo](https://raw.githubusercontent.com/djtoon/Soft-Fine-Tuning/refs/heads/main/Group%2015.png){width=50px}
An automated research/evolution loop that uses Claude Code to iteratively experiment, evaluate, and improve any project.

## How It Works

1. You define your task in `task.md` — what to test, how to run it, how to evaluate results
2. `run_research.py` launches Claude Code in a loop, each iteration reads `task.md` and performs one experiment cycle
3. Each loop: Claude reads the log (`progress.md`), plans an experiment, makes changes, runs the test, evaluates results, and logs findings
4. Next loop reads the log and builds on what was learned

```
Loop 1: Read task.md -> Plan -> Change -> Run -> Evaluate -> Log to progress.md
Loop 2: Read progress.md -> Plan -> Change -> Run -> Evaluate -> Log to progress.md
Loop 3: Read progress.md -> Plan -> Change -> Run -> Evaluate -> Log to progress.md
...
```

## Setup

1. Copy this folder into your project (or copy the files into an existing project directory)
2. Edit `task.md` — replace all `[PLACEHOLDERS]` with your project-specific instructions
3. Run:

```bash
python run_research.py
```

4. Enter how many loops to run
5. Watch Claude work

## Files

| File | What to do |
|---|---|
| `run_research.py` | The runner script. Edit `PROJECT_DIR` if your project is in a different folder. |
| `task.md` | **Edit this.** Your loop instructions — what to change, how to test, how to evaluate. |
| `progress.md` | Auto-generated by Claude. The experiment log that carries context between loops. |

## Configuration

In `run_research.py`:

- `PROJECT_DIR` — path to your project. Defaults to the folder containing the script.
- `PROMPT` — the prompt sent to Claude each loop. Default: `"Read task.md and perform the evolve research loop"`

## Requirements

- [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) installed and authenticated
- Python 3.7+

## Tips

- **Be specific in task.md** — the more detail you give about what to change, how to run, and how to evaluate, the better Claude performs
- **Start small** — run 2-3 loops first to see how it behaves, then scale up
- **Check progress.md** — if Claude is going in circles, edit task.md to redirect
- **Kill and adjust** — you can stop anytime, edit task.md or progress.md, and restart

## Example Use Cases

- Prompt engineering — iteratively improve system prompts by testing and evaluating outputs
- Config tuning — experiment with parameters, evaluate results, converge on optimal settings
- Code optimization — profile, change, benchmark, repeat
- Test coverage — find gaps, write tests, verify, repeat
- Visual QA — generate outputs, inspect for issues, fix, regenerate
- Any iterative improvement workflow where Claude can run the test and evaluate the result
